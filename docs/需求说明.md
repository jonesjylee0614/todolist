明白，砍到极简。就按你要的三件事：**现在必须做** / **未来要做** / **历史清单**。下面是只保留“必需”的功能清单（不谈技术实现）。

# 核心对象

* **任务**：`标题`（必填） + `截止日（可选）` + `备注（可选）`
* **分组**：`现在`、`未来`、`历史`

# 必需视图

1. **现在**：只显示当前必须做的任务。
2. **未来**：所有未来要做的任务（有些带截止日，有些没有）。
3. **历史**：已完成任务的清单（可按时间查看）。

# 必需操作

* **新增任务**

  * 快捷添加到`未来`（默认无截止日）。
  * 也可直接添加到`现在`（用于刚需项）。
  * 可选填截止日。
* **移动分组**

  * `未来 → 现在`（开始做）
  * `现在 → 历史`（做完）
  * `未来 → 历史`（直接完成/不再需要）
* **完成任务**：任何分组都能一键“完成”→进入`历史`。
* **编辑任务**：改标题 / 改截止日 / 改备注。
* **删除任务**：从任意分组彻底移除（与“完成”不同）。
* **搜索/过滤**（最小）：按关键词过滤当前视图。

# 必需规则（少而硬）

* **视图内排序**

  * `现在`：按添加顺序为主；有截止日的排前、越近越前。
  * `未来`：先显示有截止日（按日期升序），再显示无截止日（按添加时间）。
  * `历史`：按完成时间倒序。
* **截止日处理**

  * 逾期任务在其所在分组显式标红（不强制移动）。
  * 仅展示“今天/明天/日期”样式，避免复杂日历逻辑。
* **容量（可选但强烈建议）**

  * `现在`建议最多 **5 条**，超过时给出提示（仍允许超）。

# 必需例程（让它好用）

* **每日**（1 分钟）

  * 从`未来`拉 1–5 条到`现在`（以截止日优先）。
  * 检查 `现在` 是否超量，手动挪回`未来`。
* **每周**（5 分钟）

  * 扫一眼`历史`做回顾（可不做统计）。
  * 给`未来`里关键项补上截止日（如有需要）。

# 必需的空状态与反馈

* `现在`为空：显示“从未来拉几个进来”的按钮。
* `未来`为空：显示“添加任务”的快捷入口。
* 操作完成给轻反馈（Toast），避免复杂对话框。

# 非必需（都先不做）

* 标签/项目/优先级矩阵
* 子任务/依赖/重复任务
* 提醒/推送/日历联动/统计图表
* 多人协作/权限
* 富文本/附件

# 交互设计（简单直觉、少即是多）

## 1) 拖拽（首选交互）

* **跨栏拖拽**：

  * 未来 → 现在：任务卡片拖到“现在”列或 Tab；落下即移动
  * 现在 → 历史：拖到“历史”列或“完成”热点区
  * 未来 → 历史：同理
* **拖放反馈**：

  * 目标区域高亮，落下后出现 Toast：“已移动到现在（撤销）”
* **排序拖拽**：

  * 同一列表中可拖拽微调顺序（不破坏“截止日优先”的大规则时，可保存用户顺序为“次序权重”）

## 2) 点击/快捷操作

* **单击标题**：进入就地编辑，回车保存，Esc 取消
* **完成勾选**（可选）：在“现在/未来”都允许勾选完成 → 进入“历史”
* **快速设定截止日**：卡片上的小日历图标 → 快选“今天/明天/+1周/清除”
* **右键菜单**（或卡片三点）：

  * 移到现在 / 移到未来 / 设为完成 / 删除

## 3) 键盘快捷键（最少记忆）

* `N`：快速新增（默认进“未来”）
* `Shift+N`：新增到“现在”
* `1/2/3`：跳转 现在/未来/历史
* `Enter`：在选中任务上开始就地编辑；编辑态回车保存
* `C`：标记完成（进入历史）
* `Del`：删除（带确认）
* `/`：聚焦搜索
* `Z`：撤销上一步（或 `Ctrl+Z`）

## 4) 批量操作（不增加心智负担）

* **多选**：按 `Shift` 选连续，`Ctrl/Cmd` 选散列
* **批量移动**：多选后 `M` → 弹出“移到现在/未来/历史”
* **批量截止日**：多选后 `D` → 快选日期或清除
* **批量完成**：多选后 `C`

## 5) 触屏手势（如做移动端/平板）

* **左右滑**：在卡片上向右滑=完成；向左滑=移动（弹底部操作条：移到现在/未来/历史）
* **长按进入多选**，出现批量操作条

## 6) 顶部输入条（超快收集）

* 顶栏一个输入框：“+ 添加任务…”
* 输入后回车 → 默认进“未来”
* `!today` 或 `#now` 结尾 → 直接进“现在”
* `@2025-11-10` 或 `+3d/+1w` → 解析为截止日
  （可留到以后做，不影响 MVP）

## 7) 视觉与强调

* **逾期标红**：标题前小红点 + 截止日红色
* **今天/明天徽标**：在日期旁显示“今天/明天”
* **历史淡化**：卡片整体 60–70% 不透明度，日期灰色
* **轻动画**：拖入/完成时 150–200ms 过渡，不做复杂动效

# 用户流（3 个典型流程）

**流 1：早上开启**

1. 打开应用 → 切到“未来”
2. 多选 3–5 条 → `M` → 移到“现在”
3. 若有逾期，自动排前；必要时改截止日

**流 2：执行与收尾**

1. 在“现在”处理任务 → `C` 完成后进入“历史”
2. 若中途不做了 → 右键“移到未来”

**流 3：快速收集**

1. 顶部输入框输入标题 → 回车（进“未来”）
2. 紧急则 `Shift+N` 或加 `#now` 直接进“现在”

# 文案与微规范（保持克制）

* 主要按钮：**添加任务** / **移到现在** / **完成** / **删除**
* 空状态：

  * 现在：**今天还没有任务** → [从未来拉几个进来]
  * 未来：**没有未来任务** → [添加任务]
  * 历史：**暂无历史**（自动出现后不再显示空文案）
* Toast 示例：

  * “已移到现在（撤销）”
  * “已完成，移至历史（撤销）”
  * “已删除（撤销）”
